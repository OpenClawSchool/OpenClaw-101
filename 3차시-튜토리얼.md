## 3차시 튜토리얼

> **목적:** 3차시에는 에이전트를 개인비서에서 AI 직원으로 전환합니다. GitHub Second Brain 구축, 근무일지 자동화, 서브에이전트 생성, ClawFlows 설치까지 실습합니다.


### 목차
- [ ] 1. GitHub Second Brain 연동하기
- [ ] 2. 근무일지 자동화 설정하기
- [ ] 3. 서브에이전트 생성 규칙 등록하기
- [ ] 4. 서브에이전트 만들기
- [ ] 5. ClawFlows 설치하기
- [ ] 6. 나만의 자동화 기술 만들기




### Step 1. GitHub Second Brain 연동하기

GitHub 저장소를 에이전트 워크스페이스에 연결하고 자동 싱크를 설정합니다.

```
~/.openclaw/workspace 폴더를 내 GitHub @유저명/@레포명 에 연결하고, 변화가 생길 때마다 자동으로 싱크해줘.
(PAT 토큰: github_pat_...)
```

### Step 2. 근무일지 자동화 설정하기

매일 저녁 에이전트가 근무일지를 자동으로 작성·저장하도록 설정합니다.

```
매일저녁 9시에 크론잡으로 아래 형식으로 근무일지를 작성하고, 근무일지 폴더에 저장해줘
<형식>
# 📅 YYYY-MM-DD 근무일지

## 1. 오늘 수행한 주요 업무
- [ ] 태스크 1: (상세 설명 및 결과물 링크)
- [ ] 태스크 2: (상세 설명)

## 2. 이슈 및 해결 과정 (Troubleshooting)
- 발생한 문제점과 어떻게 해결했는지 기술 (없으면 "특이사항 없음")

## 3. 내일 예정 작업 및 제언
- 에이전트 입장에서 제안하는 후속 작업
```

### Step 3. 서브에이전트 생성 규칙 등록하기

서브에이전트를 만들 때 참고할 템플릿 파일을 등록합니다.

```
서브에이전트 새로 만들떄 "서브에이전트생성.md"파일을 참고해줘
```

### Step 4. 서브에이전트 만들기

아래 형식을 참고하여 에이전트에게 서브에이전트 생성을 요청합니다.

```
새로운 서브에이전트 만들어줘 클로드 코딩
```

> **참고:** 서브에이전트 생성 요청 시 아래 항목을 포함하면 더 정확하게 만들어집니다.
> 1. 이름/라벨  
> 2. 역할 요약  
> 3. 주요 책임 (3~5가지 태스크)  
> 4. 참조 자료 (knowledge/ 폴더 내 문서 경로)  
> 5. 필수 도구/모델

### Step 5. ClawFlows 설치하기

110개 이상의 글로벌 자동화 기술을 에이전트에게 주입합니다.

```
install clawflows
```

### Step 6. 나만의 자동화 기술 만들기

에이전트에게 맞춤형 ClawFlows 기술 생성을 요청합니다.

```
특정 웹사이트 정보를 5분마다 추적해서 엑셀로 저장하는 ClawFlows 기술을 만들어줘
```
