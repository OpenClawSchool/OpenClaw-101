# Topic Research Skill

주어진 토픽(주제)에 대해 **다각도로 자료를 수집**하고, AI 에이전트가 활용하기 좋은 **구조화된 리서치 문서**로 정리합니다.

---

## 사용 시점

- "이 주제에 대해 자료 모아줘" 또는 "~에 대해 리서치해줘"라고 요청받았을 때
- 새로운 차시/강의/프로젝트를 준비하며 배경 자료가 필요할 때
- 특정 기술·트렌드·개념에 대한 종합 보고서가 필요할 때

## 사용하지 않을 때

- 이미 정리된 문서의 편집/수정만 필요할 때
- 슬라이드 제작이 목적일 때 → `/notebooklm-slide` 사용
- 단순 질문-답변으로 해결되는 경우

---

## 워크플로우

```
1. 토픽 정의 (Define)
   ↓
2. 자료 수집 (Collect)
   ↓
3. 자료 분석 및 분류 (Analyze)
   ↓
4. 구조화 정리 (Structure)
   ↓
5. 최종 출력 (Output)
```

---

## Step 1: 토픽 정의 (Define)

사용자의 요청에서 리서치 범위를 명확히 합니다.

### 확인 항목

| 항목 | 설명 | 예시 |
|:---:|---|---|
| **핵심 토픽** | 조사할 주제 키워드 | "AI 에이전트 프레임워크 비교" |
| **리서치 목적** | 왜 이 자료가 필요한지 | 강의 준비 / 보고서 / 의사결정 |
| **범위(Scope)** | 깊이와 너비 설정 | 개론 수준 vs 기술 심층 분석 |
| **대상 청중** | 자료를 소비할 사람 | 초보자 / 개발자 / 경영진 |
| **선호 소스** | 우선 참고할 소스 유형 | 공식 문서 / 논문 / 블로그 / 뉴스 |
| **언어** | 검색·출력 언어 | 한국어 / 영어 / 혼합 |

### 정의 프롬프트 템플릿

```
다음 토픽에 대해 리서치를 진행합니다:

- 토픽: {topic}
- 목적: {purpose}
- 범위: {scope}
- 청중: {audience}
- 선호 소스: {preferred_sources}
- 출력 언어: {language}
```

> **💡 TIP:** 사용자가 간단히 "~에 대해 조사해줘"만 말한 경우, 위 항목 중 빈 부분은 합리적으로 추론하거나 사용자에게 확인합니다.

---

## Step 2: 자료 수집 (Collect)

다양한 도구를 활용하여 자료를 수집합니다.

### 수집 전략

| 순서 | 수집 방법 | 도구 | 수집 대상 |
|:---:|---|---|---|
| 1 | **웹 검색** | `search_web` | 최신 기사, 블로그, 공식 문서 |
| 2 | **URL 읽기** | `read_url_content` | 검색 결과 중 핵심 페이지 본문 |
| 3 | **로컬 파일** | `view_file`, `grep_search` | 워크스페이스 내 관련 기존 문서 |
| 4 | **Notion 검색** | `mcp_notion-mcp-server_API-post-search` | Notion 내 기존 자료 |

### 검색 키워드 생성 규칙

하나의 토픽에서 **최소 3가지 검색 각도**를 만듭니다:

```
토픽: "AI 에이전트 프레임워크"

검색 1 (정의): "AI agent framework 란 무엇인가"
검색 2 (비교): "AI agent framework 비교 2025 LangChain CrewAI"
검색 3 (활용): "AI agent framework 실제 활용 사례"
검색 4 (트렌드): "AI agent framework 최신 트렌드 2025"
검색 5 (한계): "AI agent framework 한계점 단점"
```

### 수집 품질 기준

| 기준 | 설명 |
|:---:|---|
| **최신성** | 가능하면 최근 1년 이내 자료 우선 |
| **신뢰성** | 공식 문서 > 전문 블로그 > 일반 블로그 > 커뮤니티 |
| **다양성** | 찬성·반대·중립 등 다양한 시각 포함 |
| **출처 명시** | 모든 자료에 URL 또는 출처 기록 |

---

## Step 3: 자료 분석 및 분류 (Analyze)

수집한 자료를 분석하고 카테고리별로 분류합니다.

### 분류 프레임워크

| 카테고리 | 포함 내용 | 표기 |
|:---:|---|---|
| **개념/정의** | 핵심 개념, 용어 정리 | 📖 |
| **핵심 데이터** | 통계, 수치, 벤치마크 | 📊 |
| **비교/분석** | 경쟁 제품, 장단점 비교 | ⚖️ |
| **사례/활용** | 실제 사용 사례, 튜토리얼 | 🛠️ |
| **트렌드/전망** | 미래 전망, 업계 흐름 | 🔮 |
| **우려/한계** | 문제점, 리스크, 비판 | ⚠️ |

### 핵심 인사이트 추출

각 자료에서 다음을 추출합니다:

```
- 한줄 요약 (1문장)
- 핵심 팩트 (숫자/데이터 포함)
- 인용 가능한 문구 (있는 경우)
- 출처 URL
- 신뢰도 (★~★★★★★)
```

---

## Step 4: 구조화 정리 (Structure)

분석한 자료를 최종 문서 구조로 조합합니다.

### OpenClaw 표준 지식 포맷 적용

워크스페이스의 표준 포맷(Topic-Concept-Example-Checklist)을 기반으로 작성합니다:

```markdown
# {토픽} 리서치 노트

## 메타 정보
- 조사일: {date}
- 토픽: {topic}
- 목적: {purpose}
- 범위: {scope}

## Concept (개념)
- 핵심 정의와 배경 설명

## Key Findings (핵심 발견)
### 📊 데이터 & 통계
- 주요 수치와 벤치마크

### ⚖️ 비교 분석
- 주요 대안/경쟁 비교표

### 🛠️ 활용 사례
- 실제 적용 사례 정리

### 🔮 트렌드 & 전망
- 앞으로의 방향성

### ⚠️ 한계 & 고려사항
- 주의할 점, 리스크

## Example (사례)
- 에이전트가 참고할 구체적 예시

## Sources (출처)
- 참고한 모든 URL과 자료 목록

## Checklist (검증)
- [ ] 핵심 개념이 명확히 정의되었는가?
- [ ] 데이터/수치의 출처가 명시되어 있는가?
- [ ] 다양한 시각(긍정/비판)이 포함되었는가?
- [ ] 에이전트가 활용 가능한 구조인가?
```

---

## Step 5: 최종 출력 (Output)

### 파일명 규칙

`{토픽키워드}_research.md` 형식으로 저장합니다:

| 토픽 | 출력 파일 |
|:---:|:---:|
| AI 에이전트 프레임워크 | `ai_agent_framework_research.md` |
| 프롬프트 엔지니어링 | `prompt_engineering_research.md` |

### 출력 위치

사용자가 지정한 위치 또는 프로젝트 루트에 저장합니다.

---

## 사용자 프롬프트 예시

### 기본 요청
```
"AI 에이전트에 대해 자료 모아줘"
```

### 상세 요청
```
"프롬프트 엔지니어링 기법에 대해 리서치해줘.
- 목적: 2차시 강의 자료 준비
- 범위: 초보자가 바로 쓸 수 있는 실전 기법 중심
- 청중: 비개발자
- 한국어와 영어 소스 모두 포함"
```

### 부분 요청
```
"이 토픽에 대해 최신 트렌드만 조사해줘"
"경쟁 제품 비교표만 만들어줘"
"이 주제에 대한 비판적 시각 자료를 찾아줘"
```

## 품질 체크리스트

- [ ] **최소 5개 이상의 소스**에서 자료를 수집했는가?
- [ ] **다양한 각도** (정의/비교/사례/전망/한계)가 포함되었는가?
- [ ] 모든 핵심 데이터에 **출처 URL**이 명시되었는가?
- [ ] **최신 자료** (최근 1년 이내)가 우선 포함되었는가?
- [ ] **OpenClaw 표준 포맷** (Concept-Example-Checklist)이 적용되었는가?
- [ ] 에이전트가 **지식 문서로 바로 활용**할 수 있는 구조인가?
- [ ] 사용자가 요청한 **범위와 목적**에 부합하는가?
